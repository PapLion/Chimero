## Top Findings

1. Weight is the only tracker that currently meets the specialized-schema bar. It has `entries + entry_weight + tracker_goals`, specialized IPC/service handlers, Weight read models, real detail/chart data, and atomic Weight CRUD. Remaining gaps are honest but not blockers for its current simple scope.

2. Mood is not equivalent to Weight. Mood has a seeded tracker, Quick Entry, a mood widget, daily aggregate endpoint, and generic CRUD, but it still lacks a Mood-specific schema/service and has a 1-10 UI versus `max: 5` seed/config mismatch.

3. Exercise, Gaming, Diet/Meals, and Social are supposed to be deep trackers in client context, but current implementation is mostly generic scaffolding plus a few specialized UI helpers. They should stay PARTIAL until each has a real contract-backed schema/read model.

4. Books, Media/TV, Tasks, and Hydration have useful generic surfaces, but generic note/value persistence is not DONE for structured tracker requirements. Their current status is generic/PARTIAL unless the client confirms that simple generic behavior is enough.

5. Sleep, Meditation, Health, and Vitamins/Medications are client-discovery/doc-only in the audited line range. They should not be treated as implemented just because dynamic custom trackers and generic detail tabs exist.
